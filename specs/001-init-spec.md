## 1. 产品概述

### 1.1 产品名称

AQingCode — 面向开发者的 AI 原生桌面代码编辑器

### 1.2 产品定位

AQingCode 是一款以 AI 为核心驱动力的轻量级桌面代码编辑器，整体界面风格对标 VS Code，在提供完整代码编辑能力的基础上，深度集成 AI 对话、AI 代码补全、AI Agent 自主任务执行等能力。采用 Tauri 2.0 构建，兼顾跨平台性能与轻量化体积。

### 1.3 目标用户

- 希望借助 AI 提升编码效率的个人开发者
- 需要轻量级本地 AI 编程助手的技术人员
- 对 Cursor / Windsurf 等产品感兴趣、希望使用国产替代的开发者

### 1.4 核心价值主张

> "让 AI 成为你的结对编程伙伴，而不只是一个补全工具"

- AI 内联补全：写代码时实时获得多行智能建议
- AI Chat 面板：与代码上下文深度联动的对话式编程
- AI Agent 模式：自主完成多文件编辑、终端执行等复杂任务
- 多模型支持：自由接入 Claude、GPT-4o、通义千问等主流模型

## 2. 功能需求

### 2.1 基础编辑器功能

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 打开文件夹 | 通过菜单「文件 → 打开文件夹」选择本地目录作为工作空间 | P0 |
| 目录树展示 | 左侧递归展示文件夹与文件，支持展开/折叠 | P0 |
| 文件单击打开 | 单击文件节点，右侧以 Tab 页打开文件 | P0 |
| 多 Tab 页管理 | 支持同时打开多个文件，Tab 标签展示 | P0 |
| 文件编辑与保存 | 集成 Monaco Editor，`Ctrl+S` 保存，未保存显示「●」标记 | P0 |
| 关闭 Tab | 点击关闭按钮关闭文件，有未保存修改时弹出确认 | P0 |
| 右键菜单 | 右键导航树文件/文件夹，提供「复制」「重命名」「删除」操作 | P0 |
| 删除确认 | 删除操作前弹出二次确认对话框 | P0 |
| 工作空间持久化 | 记录上次打开的工作空间，下次启动自动恢复 | P1 |

### 2.2 AI 代码补全（Inline Completion）

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 内联幽灵文本补全 | 编辑器中实时显示灰色幽灵文本（Ghost Text）形式的 AI 建议 | P0 |
| 多行代码补全 | 支持一次性补全多行代码块，不仅限于单行 | P0 |
| Tab 键接受补全 | 按 `Tab` 接受当前 AI 建议，`Esc` 取消 | P0 |
| 逐词接受 | `Ctrl+→` 逐词接受补全内容 | P1 |
| 补全触发控制 | 支持手动触发（`Alt+\`）和自动触发两种模式 | P1 |

### 2.3 AI Chat 对话面板

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 侧边 Chat 面板 | 右侧或底部可展开的 AI 对话面板，支持多轮对话 | P0 |
| 代码上下文感知 | Chat 自动感知当前打开文件和光标位置作为上下文 | P0 |
| `@文件` 引用 | 在对话中输入 `@文件名` 引用特定文件内容 | P0 |
| 选中代码发送 | 选中代码后右键或快捷键将其发送到 Chat 面板 | P0 |
| Apply 到编辑器 | AI 回复中的代码块支持一键 Apply 到对应文件 | P0 |
| Diff 预览 | Apply 前展示变更 Diff，用户确认后再写入 | P1 |
| 斜杠命令 | 支持 `/explain`（解释）、`/fix`（修复）、`/refactor`（重构）、`/tests`（生成测试）等快捷命令 | P1 |
| 对话历史 | 保留当前会话的对话历史，支持清空 | P1 |

### 2.4 AI 代码操作（右键菜单 & 快捷操作）

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 解释代码 | 选中代码 → 右键 → 「AI 解释」，在 Chat 面板展示解释 | P0 |
| 修复问题 | 选中代码 → 右键 → 「AI 修复」，AI 给出修复建议并可 Apply | P0 |
| 重构代码 | 选中代码 → 右键 → 「AI 重构」，AI 给出重构方案 | P1 |
| 生成单元测试 | 选中函数/类 → 右键 → 「生成测试」，AI 自动生成测试代码 | P1 |
| 生成注释 | 选中代码 → 右键 → 「生成注释」，AI 自动补全文档注释 | P1 |

### 2.5 AI Agent 模式

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| Agent 模式入口 | Chat 面板中切换到「Agent 模式」，AI 可自主执行多步任务 | P1 |
| 多文件自主编辑 | Agent 可读取、创建、修改多个文件完成复杂任务 | P1 |
| 终端命令执行 | Agent 可在内置终端执行命令（如安装依赖、运行测试） | P1 |
| 任务进度展示 | 展示 Agent 当前执行步骤和操作日志，用户可随时中断 | P1 |
| 操作确认机制 | 文件写入、命令执行等高风险操作需用户确认后执行 | P1 |

### 2.6 AI 模型配置

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| API Key 配置 | 在设置面板中配置 OpenAI / Anthropic / 通义千问等 API Key | P0 |
| 多模型切换 | 支持在 GPT-4o、Claude 3.5/3.7、通义千问等模型间切换 | P0 |
| 自定义 Base URL | 支持配置自定义 API 地址，兼容 OpenAI 协议的第三方服务 | P1 |
| 补全模型与对话模型分离 | 可分别为内联补全和 Chat 对话配置不同模型 | P1 |
| 模型参数配置 | 支持配置 Temperature、Max Tokens 等基础参数 | P2 |

### 2.7 上下文管理

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 当前文件上下文 | AI 默认以当前打开文件为上下文 | P0 |
| `@文件` 手动引用 | 在 Chat 中通过 `@` 符号引用工作空间内任意文件 | P0 |
| 工作空间索引 | 对工作空间代码进行本地向量化索引，支持语义级代码检索 | P2 |
| `@代码库` 全局问答 | 基于工作空间索引，支持跨文件的代码库级别问答 | P2 |

### 2.8 内置终端

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 集成终端面板 | 底部集成 xterm.js 终端，支持 Shell 命令执行 | P1 |
| 多终端 Tab | 支持同时开启多个终端会话 | P2 |
| AI 终端辅助 | 在终端中输入 `#` 前缀，AI 将自然语言转换为 Shell 命令 | P2 |

## 3. 技术架构

### 3.1 技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 桌面框架 | Tauri 2.0 | 跨平台桌面应用壳，Rust 后端，包体积小 |
| 前端框架 | React 19 | UI 组件构建 |
| 语言 | TypeScript | 前端类型安全 |
| 代码编辑器 | Monaco Editor | VS Code 同款编辑器内核，语法高亮与智能提示 |
| 样式 | Tailwind CSS | 原子化 CSS 快速布局 |
| 终端 | xterm.js | 集成内置终端 |
| 后端逻辑 | Rust (Tauri Commands) | 文件系统读写、目录遍历、进程管理 |
| AI 接入 | OpenAI SDK / Anthropic SDK | 流式对话、补全接口调用 |

### 3.2 架构分层

```
┌──────────────────────────────────────────────────────────┐
│                      React 前端                           │
│  ┌──────────────┐  ┌─────────────────┐  ┌─────────────┐ │
│  │  文件资源管理器  │  │  Monaco Editor  │  │  AI Chat    │ │
│  │  （导航树）    │  │  （编辑区+Tab）  │  │  面板       │ │
│  └──────────────┘  └─────────────────┘  └─────────────┘ │

│  │  xterm.js    │  │  AI Inline Completion Provider  │   │
│  │  （终端）     │  │  （Monaco 补全扩展）              │   │
│  └──────────────┘  └─────────────────────────────────┘   │
│                  ↕ Tauri invoke / event                   │
├──────────────────────────────────────────────────────────┤
│                   Rust 后端 (Tauri)                       │
│  文件读写 / 目录遍历 / 重命名 / 删除 / 进程管理 / 配置存储   │
├──────────────────────────────────────────────────────────┤

└──────────────────────────────────────────────────────────┘
```

## 5. 非功能性需求

| 类别 | 要求 |
|------|------|
| 性能 | 启动时间 < 3 秒；AI 补全响应延迟 < 500ms（流式输出首 token）|
| 平台支持 | Windows 10+、macOS 12+、主流 Linux 发行版 |
| 安装包体积 | 目标 < 30MB（Tauri 优势）|
| 安全性 | API Key 本地加密存储，不上传至任何服务器；文件操作权限仅限工作空间目录 |
| 离线可用 | 基础编辑功能（无 AI）在无网络环境下完整可用 |
| 隐私 | 代码内容仅在用户主动触发 AI 功能时发送至 AI 服务，不做后台上传 |

## 8. 验收标准

### 基础编辑器
- [ ] 可通过「文件」菜单选择并打开本地文件夹作为工作空间
- [ ] 左侧导航树正确展示目录结构，支持展开/折叠
- [ ] 单击文件节点，右侧以 Tab 页打开文件并展示内容
- [ ] Monaco Editor 对常见语言（TS/JS/Rust/JSON/HTML/CSS）提供语法高亮
- [ ] `Ctrl+S` 保存文件，未保存状态有「●」标记
- [ ] 右键菜单提供「复制」「重命名」「删除」三项操作且功能正常

### AI 功能
- [ ] 在设置中配置 API Key 后，编辑器中出现 AI 内联补全幽灵文本
- [ ] 按 `Tab` 可接受 AI 补全建议，`Esc` 可取消
- [ ] AI Chat 面板可正常发送消息并流式接收回复
- [ ] Chat 中输入 `@文件名` 可引用工作空间文件内容
- [ ] AI 回复中的代码块有「Apply」按钮，点击后代码写入对应文件
- [ ] 选中代码后右键可触发「AI 解释」「AI 修复」操作
- [ ] 状态栏显示当前使用的 AI 模型名称
- [ ] API Key 以加密形式存储，不以明文出现在配置文件中
